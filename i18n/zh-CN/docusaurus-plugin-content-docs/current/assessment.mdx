# 流畅度评估工具

import Assessment from '@site/src/components/FluencyModel/Assessment';

AI 编码流畅度评估工具帮助您规划每个维度的当前和目标流畅度等级，并识别实现目标所需的关键投入。

灵感来源于 [ThoughtWorks 数字流畅度模型](https://www.thoughtworks.com/digital-fluency/digital-fluency-model)，本工具提供：

- **当前状态评估** - 识别您的团队目前所处位置
- **目标设定** - 定义您希望达到的水平
- **投资路线图** - 获取每个阶段的具体建议
- **进度追踪** - 您的选择会自动保存

## 工作原理

### 第一步：选择维度
点击五个构建模块中的任意一个开始评估。每个维度代表 AI 辅助开发的一个关键方面。

### 第二步：评估当前等级
为每个维度选择最能描述您团队当前状态的等级。请诚实作答——这是为了了解您的现状，而非期望状态。

### 第三步：设定目标等级
选择您的目标等级。这应该是雄心勃勃但在您的规划周期内（通常是 6-18 个月）可实现的。

### 第四步：查看投资建议
工具将显示从当前等级进阶到目标等级所需的建议投入。

### 第五步：生成您的模式
完成所有维度的评估后，查看您完整的流畅度模式——当前状态、目标和差距的可视化呈现。

## 核心原则

### 流畅度是愿景性的
期望的流畅度等级因能力、组织甚至同一组织的不同部门而异。

### 非均匀进展
团队不需要在所有维度上处于相同等级。某些维度可能需要先进步，其他维度才能跟进。

### 渐进式步骤
每个等级都建立在前一个等级之上。建议每次只提升一个等级。

### 因地制宜
正确的等级取决于您团队的目标、约束和业务背景。没有放之四海而皆准的答案。

## 时间框架

基于典型的组织变革模式：

| 转变 | 典型时间框架 |
|------|-------------|
| 等级 1 → 2 | 3-6 个月 |
| 等级 2 → 3 | 6-12 个月 |
| 等级 3 → 4 | 12-18 个月 |
| 等级 4 → 5 | 18-36 个月 |

这些是指导原则，而非硬性规定。您的实际时间线将取决于起点、资源和组织准备度。
