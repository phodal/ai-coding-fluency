# Awareness (认识/意识唤醒)

Teams begin to recognize AI coding tools' existence and potential, but usage remains scattered and passive.

## Description

At this stage, developers primarily interact with AI through Q&A, treating it as a smarter search engine. AI tools exist as standalone chat interfaces with no integration into existing development toolchains. Quality assurance relies entirely on manual review, and each conversation is independent with no context memory.

This is the starting point for most teams encountering AI coding tools.

## Key Signals

- ✓ Developers start using ChatGPT and similar tools for technical queries
- ✓ Team maintains cautious attitude toward AI-generated code
- ✓ AI usage depends entirely on individual initiative

## Key Investments

To progress from this level, teams should focus on:

- → Organize internal sharing sessions on AI coding tools
- → Establish basic guidelines and best practices for AI usage
- → Encourage team members to experiment with different AI tools

## Metrics

Track these indicators to measure progress:

- 📊 Team awareness of AI tools
- 📊 AI tool trial rate
- 📊 Frequency of internal sharing and discussions

## Next Level

Once your team has established basic awareness and experimentation with AI tools, you're ready to move to [Assisted Coding](/docs/levels/assisted-coding), where AI becomes integrated into daily development workflows.
